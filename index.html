Rules & Guidelines

To ensure a fair, competitive, and fun event for everyone, please adhere to the following rules. Read them carefully.

What's Allowed

  • Frontend Frameworks

    React, Vue.js, Svelte, Angular, or plain HTML/CSS/JS.

  • Hosting Platforms

    Vercel, Netlify, Firebase Hosting, GitHub Pages, Render.

  • Backend Services

    Node.js, Firebase (Auth, Firestore), Supabase, PocketBase.

  • Open-Source Libraries

    You may use any open-source libraries (e.g., from npm).

What's Forbidden

  • AI Code Generators

    Tools like ,GitHub Spark, Co pilot, AI Studio , Firebase Studio , Vercel , Figma , WIX , Lovable are disabled to ensure original work.

  • Pre-built Templates

    Using existing templates or themes is not permitted.

  • Existing Codebases

    All code must be written during the hackathon.

  • Paid Services

    Projects must not require paid plans or credit cards to run.

Submission Requirements

  • A publicly accessible, live deployed URL.
  • A public GitHub repository link.
  • A detailed README.md file explaining your project.
  • A short, unlisted YouTube video demo (max 3 mins).

Judging Criteria

  • Innovation & Creativity (25%)

    How original and impactful is the idea?

  • Technical Implementation (25%)

    How well-built and functional is the project?

  • Design & UX (20%)

    Is the interface intuitive and visually appealing?

  • Code Quality & Docs (15%)

    Is the code clean, organized, and well-documented?

  • Presentation (15%)

    How effectively is the project demoed?

Step-by-Step Guides

Follow these tutorials to get your project from zero to deployed.

  1. Create a New Repository on GitHub

    Start by creating a new public repository. This will be the home for your project's code.

    Repository name: your-project-name
    Description: A great description of your project.
    ☑️ Public
    ☑️ Add a README file
    ☑️ Add .gitignore: Node

  2. Clone the Repository to Your Machine

    Use the `git clone` command to create a local copy of your new repository.

    git clone https://github.com/YOUR_USERNAME/your-project-name.git cd your-project-name

  3. Daily Git Workflow

    Follow this commit cycle regularly to save your progress and keep your repository up-to-date.

    # Check the status of your changes git status # Add all new and changed files to staging git add . # Commit your changes with a descriptive message git commit -m "feat: Implement user authentication" # Push your commits to GitHub git push origin main

  1. Install Node.js & npm

    React development requires Node.js. Download the LTS version from nodejs.org.

  2. Create a New React Application

    Use Create React App to quickly scaffold a new project with a modern build setup.

    npx create-react-app .

  3. Essential Development Commands

    These commands will be your best friends during development.

    # Starts the local development server npm start # Installs a new dependency from npm npm install package-name # Bundles the app for production npm run build

  1. Deploy to Vercel via GitHub

    The easiest way to deploy is by linking your GitHub account to Vercel for continuous deployment.

    1. Visit vercel.com and sign up with GitHub.
    2. Click "Add New... > Project" and import your GitHub repository.
    3. Vercel will auto-detect that it's a React app. Keep the default settings.
    4. Click "Deploy". Your site will be live in minutes!

  2. Deploy to Firebase Hosting

    Firebase is another excellent option, especially if you're using other Firebase services.

    # Install Firebase CLI globally npm install -g firebase-tools # Login and initialize hosting firebase login firebase init hosting # Build your React project first npm run build # Deploy to Firebase firebase deploy
